# Security Policy

## Reporting a Vulnerability

Please do not disclose vulnerabilities, credential leaks, or secret-handling mistakes in a public GitHub issue.

For now, report security issues privately to the project maintainer and include:

- a description of the issue
- affected files, routes, or flows
- reproduction steps if applicable
- potential impact
- whether any credentials or production systems may be affected

If you are already in direct contact with the maintainer, use that private channel. A dedicated security contact will be added before broader public rollout.

## What to Avoid in Public Reports

- API keys
- database credentials
- Supabase service keys
- production URLs with active tokens
- proof-of-concept exploit details that materially increase risk before a fix exists

## Response Goals

As the project matures, this policy will be updated with target response windows and disclosure guidance.
